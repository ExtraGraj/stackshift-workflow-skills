# 0.2.2 — Workflow and Architecture Fixes

## Protocol Directory Cleanup

Renamed overview documents inside two multi-file protocol directories from `SKILL.md` to `overview.md`. The name `SKILL.md` is reserved for agentic skill router entry points; using it inside a protocol directory caused ambiguity about whether these files were loadable skill roots. Both files acted as protocol overviews and are now named accordingly.

- `protocols/_template/SKILL.md` → `protocols/_template/overview.md`
- `protocols/modal-sheet/SKILL.md` → `protocols/modal-sheet/overview.md`

No registry entries or cross-references were affected. The `_registry.json` entry for `modal-sheet` uses `"dir"` (not `"file"`), and the `_template/` directory is referenced by path only.

## Step Ordering — Honest Architecture Documentation

Removed language in `SKILL.md` and `README.md` that implied the router prevents files from being loaded before they are needed. Step ordering is enforced by instruction, not by file-access control. Updated the workflow table note and lookup router section header to reflect this accurately. Updated the "On-Demand Loading Architecture" section in `README.md` to "Lookup-Table Architecture" with accurate descriptions.

## Step 4 — Fix Mode, Fidelity Check, and Screenshot Refs

### Fix/Rebuild Mode Decision Table

Added a "Modification vs. Full Generation" decision table before sub-step 4a. Fix and rebuild tasks require `--mode body-only` with an HTML reference; full generation uses the default mode. Both modes require FORGE NOTES and postcondition validation.

### Anti-Slop Fidelity Check

Added a pre-generation checklist to Step 4d that activates when any `--refs` file is HTML or TSX. Requires the agent to verify padding/margin values, background context, decorative elements, icon container dimensions, and button style against the reference before writing output, and report findings in FORGE NOTES.

### Screenshot Refs Must Use `--refs`

Clarified in the ref-selection rules table that reference screenshots must be passed as `--refs path/to/screenshot.png`. The `+IMAGE` modifier only fires when an image file appears in `--refs`; relying on AI vision context alone does not activate it.
