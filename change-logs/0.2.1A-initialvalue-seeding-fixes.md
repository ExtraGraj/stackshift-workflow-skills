# 0.2.1A — InitialValue Seeding Fixes

**Released:** 2026-05-06

---

## Summary

Two high-priority fixes to `seeds/initialValue-seeding.md`. No CLI changes. No breaking changes.

---

## A-1 · Complete file example added to seeding strategy

**Affects:** `skills/stackshift-core/seeds/initialValue-seeding.md`

Added a **"Complete file example"** section between "How to execute" and "Handling complex data". The example shows the canonical `initialValue/index.ts` structure with:

- All required imports from `@webriq-pagebuilder/sanity-plugin-schema-default`
- The `export default { ... }` shape
- All three default helpers in use: `defaultButton`, `defaultBlockContent`, `defaultArrayOfObjects`
- A mix of field types: string, Portable Text with marks and list items, button/CTA, string array, and array of objects

A helper reference table is included immediately after the example.

This example is loaded when no existing `initialValue/` files are present in the project, preventing agents from producing files with wrong export shapes, missing imports, or plain strings where Portable Text is required.

---

## A-2 · Seeding sources enforcement added

**Affects:** `skills/stackshift-core/seeds/initialValue-seeding.md`

Added **Step 0 — Identify all seeding sources before writing** to the "How to execute" section. This step forces the agent to:

1. Read the reference HTML/component/file before writing any entry (no proceeding from memory).
2. Check for co-located data/config files (`data/<section>.ts`, `config/<section>.ts`, `_data.ts`, `<section>.data.ts`) and use them as the primary seed source when found.
3. Use generic-but-realistic placeholder text only when neither a reference nor a data file exists.

Updated **"Text conventions"** to add two enforcement rules:
- Literal content from a reference HTML or data file is mandatory — no substitution or paraphrasing.
- Config/data files take precedence over reference HTML as the authoritative seed source.

---

## Seed enforcement elevated to required gate

**Affects:** `skills/stackshift-core/workflow/2-section-schema.md`

The seed strategy check was previously listed under "Recommended actions" in Step 2, meaning an agent could skip it. It is now a dedicated **"Seed gate"** section placed before "Recommended actions" with required semantics:

- If `.stackshift/installed.json` has a `seed` field, the agent must load the matching strategy from `seeds/<file>` before writing any `initialValue/` file.
- If no `seed` field is present, the gate is a no-op.
- The gate is skipped entirely when the current task does not touch `initialValue/`.

This means the active seeding strategy is now applied automatically whenever `initialValue/` files are being written — no verbal instruction from the user required.
