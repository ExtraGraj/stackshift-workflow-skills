# 0.2.3B — UI Forge 0.2.7–0.2.7E Sync

**Released:** 2026-05-06

---

## Summary

This release synchronizes StackShift documentation and protocols with five UI Forge sub-releases (0.2.7 through 0.2.7E) that landed between 2026-05-05 and 2026-05-06. Several changes directly impact the StackShift ↔ UI Forge paired-mode contract, variant generation flow, and bootstrap behavior. All five StackShift companion-facing files are updated to reflect the new contract terms and terminology.

---

## D-1 · UI Forge 0.2.7–0.2.7E compatibility sync

**Affects:** `protocols/paired-mode-contract.md`, `workflow/4-variants.md`, `bootstrap/install.md`, `references/versions.md`

### UI Forge 0.2.7 — `design/standards/` auto-creation and theme forcing

**Change:** UI Forge's `scan.js` now auto-creates `design/standards/` directory if missing. The `--theme stackshift` flag now forces `isStackShift: true` in design-arch.json. UI Forge's `install` step wires `variant-router` directly into `design-arch.json`.

**StackShift impact:**
- Bootstrap Step 6f can now safely assume the directory exists after Step 6c completes
- `references/versions.md` updated with new 0.2.1–0.2.2 / ≥0.2.7 compatibility row

### UI Forge 0.2.7A — Named export validation fix

**Change:** `validate-contract.js` false-positive fixed. Prior to 0.2.7A, a named re-export after the default export (e.g., `export { MySection_X }`) was flagged as a violation. From 0.2.7A onward, when `.stackshift/installed.json` is present, `validate-contract.js` auto-detects paired mode and exempts exactly one named export matching the default export name.

**StackShift impact:**
- New subsection added to `protocols/paired-mode-contract.md` documenting the "Named export exception"
- Clarifies that this is a benign outcome and requires no StackShift changes
- Makes the contract term auditable in one place

### UI Forge 0.2.7B — `--mode body-only` FORGE NOTES placement

**Change:** `--mode body-only` now overrides the "variant file begins with FORGE NOTES" rule. In body-only mode, the `// FORGE NOTES` header is placed immediately after the last import statement, not at the beginning of the file.

**StackShift impact:**
- `workflow/4-variants.md` postcondition updated from flat rule to conditional:
  - **Full mode (default):** file begins with `// FORGE NOTES`
  - **Body-only mode (`--mode body-only`):** `// FORGE NOTES` appears immediately after the last import statement (UI Forge ≥ 0.2.7B)

### UI Forge 0.2.7C — UTF-8 stdout fix for Windows redirects

**Change:** Fixed UTF-8 handling when output is redirected to a file on Windows (`node invoke.js ... > file.md`).

**StackShift impact:**
- Informational only; no StackShift file changes required
- Improves reliability of variant generation in Windows CLI environments

### UI Forge 0.2.7D — Modification/fix mode and `+IMAGE` standard

**Change:** Modification and fix modes documented and formalized. New anti-slop fidelity checklist (5 items) added. `+IMAGE` modifier fallback behavior documented. New UI Forge reference file: `references/standards/nextjs-image.md` (Next.js + Sanity image best practices).

**StackShift impact:**
- `workflow/4-variants.md` already documents `+IMAGE` correctly at line 284 — no changes needed
- No StackShift protocol changes required
- Project teams can reference the new `nextjs-image.md` standard in UI Forge skill when implementing image fields

### UI Forge 0.2.7E — README restructure and CLI terminology

**Change:** UI Forge README restructured for clarity. CLAUDE.md condensed. SKILL.md updated to use "Slash Commands (Agentic CLI)" instead of "Claude Code" (inclusive of all platforms). `detect.js` now symlink-aware.

**StackShift impact:**
- `workflow/4-variants.md` invocation section labels updated for cross-platform accuracy:
  - Old: `Claude Code (preferred when .claude/ directory exists):`
  - New: `Slash Commands / Agentic CLI (preferred — Claude Code, AntiGravity, or any agentic platform with .claude/ directory):`
  - Old: `General agents / UI Forge < 0.1.9:`
  - New: `Universal agent invocation (all platforms, or UI Forge < 0.1.9):`
- Terminology now aligns with UI Forge 0.2.7E's inclusive CLI framing
