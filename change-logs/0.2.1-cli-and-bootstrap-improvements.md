# 0.2.1 — CLI & Bootstrap Improvements

**Released:** 2026-05-06

---

## Summary

This release addresses two install-flow bugs in the CLI bootstrap flow, expands the agentic platform selection in the CLI, and improves custom protocol selection UI.

---

## B-2 · Bootstrap artifacts moved to `.stackshift/`

**Affects:** `skills/stackshift-core/bootstrap/install.md`, `bootstrap/modes.md`, `SKILL.md`, `workflow/1-schema-fields.md`, `workflow/2-section-schema.md`, `workflow/4-variants.md`, `workflow/checklist.md`, `protocols/auto-verify-hook.md`, `protocols/claude-design-handoff.md`

All bootstrap-managed project infrastructure is now co-located under `.stackshift/`:

| Artifact | Old location | New location |
|----------|-------------|-------------|
| Protocol registry | `/docs/protocol/_registry.json` | `.stackshift/protocol/_registry.json` |
| Protocol template | `/docs/protocol/_template/` | `.stackshift/protocol/_template/` |
| Materialized protocols | `/docs/protocol/<id>` | `.stackshift/protocol/<id>` |
| Custom references | `/docs/references/` | `.stackshift/references/` |

**Not changed:** `design/standards/` (UI Forge files), `design/design-arch.json`, `.forgeignore`, `.claude/settings.json` — these remain at their existing locations.

`.stackshift/` is the machine-managed state directory. Protocol files in `.stackshift/protocol/` are team-customizable (edit freely; project copies take precedence over skill defaults at lookup time).

## B-1 · Pre-bootstrap marker + `--bootstrap` flag

**Affects:** `cli/src/install.ts`, `cli/src/writer.ts`, `cli/src/flags.ts`, `skills/stackshift-core/bootstrap/install.md`, `skills/stackshift-core/SKILL.md`

### Pre-bootstrap marker (Option 1)

On fresh project installs (no existing `installed.json`), the CLI now writes:

```json
{
  "skillVersion": "0.2.1",
  "installedAt": "...",
  "mode": "recommended",
  "protocols": [...],
  "bootstrapRequired": true
}
```

`SKILL.md` Section 0 bootstrap check now treats `bootstrapRequired: true` as "not bootstrapped yet" and runs the full interactive bootstrap flow. After bootstrap completes, the agent removes the `bootstrapRequired` field and writes the real state.

On re-installs of already-bootstrapped projects (`bootstrapRequired` absent), `bootstrapRequired` is NOT re-added — the completed state is preserved.

### `--bootstrap` flag (Option 3)

```bash
# Interactive — shows prompts, then materializes
npx @extragraj/stackshift-skills init --bootstrap

# Non-interactive — no prompts, uses flags + defaults
npx @extragraj/stackshift-skills init --tier recommended --bootstrap --no-interactive
```

When `--bootstrap` is specified, the CLI runs protocol materialization after installing skill files — whether or not `--no-interactive` is passed:

- Copies selected protocols to `.stackshift/protocol/`
- Writes `.stackshift/protocol/_registry.json`
- Copies `_template/` to `.stackshift/protocol/_template/`
- Creates `.stackshift/references/README.md`
- Creates `design/standards/stackshift-ui.md`
- Creates `design/standards/brand.md` (only when `brand` protocol is selected)
- Writes `.forgeignore` (only if not already present)

`bootstrapRequired` is NOT written when `--bootstrap` is used — the project is considered bootstrapped by the CLI. UI Forge integration steps (Steps 6c–6h, 7b) still require the agent on first invocation.

## CLI · Expanded platform selection

**Affects:** `cli/src/prompts.ts`, `cli/src/flags.ts`, `cli/src/writer.ts`, `cli/src/repair.ts`, `cli/src/install.ts`

Platform selection now includes all major agentic platforms:

| Platform | Label | Project skill root | Global skill root |
|----------|-------|-------------------|------------------|
| `claude` | Claude Code | `.claude/skills/` | `~/.claude/skills/` |
| `copilot` | GitHub Copilot (Agent Mode) | `.github/skills/` | `~/.copilot/skills/` |
| `agents` | OpenAI Codex / Universal Agents | `.agents/skills/` | `~/.agents/skills/` |
| `gemini` | Google Gemini (Antigravity) | `.agents/skills/` | `~/.gemini/antigravity/skills/` |
| `cursor` | Cursor IDE | `.cursor/skills/` | `~/.cursor/skills/` |

**Default platform changed from `claude` to `agents`** (universal — works with all general-purpose agents).

`repair` now scans all five platform directories for bundles and seed folders (previously only scanned `agents` and `claude`). Path deduplication prevents double-writes when two platforms share the same directory (e.g. `agents` and `gemini` both use `.agents/` for project scope).

## CLI · Custom protocol selection hint

**Affects:** `cli/src/prompts.ts`

In the custom protocol multiselect, each protocol's hint now includes its tier label:

```
[recommended] Check local and base-package field factories before creating any…
[optional] Registers a project brand document so UI Forge applies voice, palette…
```

Summaries are truncated to 72 characters to keep the list readable.
