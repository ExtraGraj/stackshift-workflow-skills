**Date:** 2026-05-06

## Problem

When a user re-ran `npx stackshift init` with a narrower protocol tier (e.g. downgrading from `full` to `required`), the CLI materialized the new selection into `.stackshift/protocols/` but left previously materialized files for removed protocols on disk. Because the skill's lookup always checks `.stackshift/protocols/` first, those stale files continued to load and take precedence over the bundled defaults â€” silently.

The `repair` command also had no awareness of `.stackshift/protocols/`: it only cleaned up multi-tier `stackshift-protocols-*` skill bundle folders, not orphaned materialized protocol files.

## Changes

### `cli/src/writer.ts` â€” `runBootstrapMaterialization()`

- Added `removed: string[]` and `updated: string[]` to the `BootstrapResult` interface.
- Introduced `PROTOCOL_DESIGN_STANDARDS` map â€” declares which `design/standards/` files are seeded conditionally per protocol (`brand` â†’ `brand.md`). Extend this map when future protocols seed design standards files.
- On reinstall detection (`installed.json` exists with `materializationDone: true`):
  - Computes the diff between the new protocol selection and all known protocols.
  - **Removes** materialized files/dirs from `.stackshift/protocols/` for protocols no longer in the new selection (`toRemove`).
  - **Removes** `design/standards/` files seeded by those removed protocols (e.g. `design/standards/brand.md` when `brand` is deselected).
  - **Skips** (preserves with user edits) protocols present in both old and new selection (`toKeep`).
  - **Materializes** protocols in the new selection not yet on disk (`toAdd`).
  - **Updates** `.stackshift/protocols/_registry.json` to reflect the new materialized list, preserving the `custom` array.

### `cli/src/repair.ts` â€” `repair()`

- Added a new **Materialized Protocol Reconciliation** section that runs after seed repair.
- Reads `.stackshift/installed.json` to determine the recorded protocol list.
- Loads the full protocol registry to identify all known protocol files.
- **Removes** orphaned files in `.stackshift/protocols/`: known protocols on disk that are NOT in the recorded list.
- **Removes** orphaned `design/standards/` files: protocol-conditional standards files whose protocol is no longer recorded (uses the same `PROTOCOL_DESIGN_STANDARDS` mapping as `writer.ts`).
- **Restores** missing files: recorded protocols NOT on disk (re-copied from the skill).
- Reports the reconciliation result in the repair summary.

### `cli/src/install.ts`

- Updated materialization output notes to display `Updated` and `Removed (stale)` entries from `BootstrapResult`.
- Applied to both interactive and non-interactive install paths.

### `skills/stackshift-core/bootstrap/install.md` â€” Step 5A

- Added a **Reinstall / Mode Change Path** section before the materialization loop.
- Documents the AI agent's expected behavior when protocol files already exist on disk from a previous bootstrap run: detect stale protocols, prompt the user, and remove on confirmation.

## Upgrade notes

No migration required. On next `npx stackshift init` run in an existing project, stale protocols are automatically removed and `.stackshift/protocols/_registry.json` is updated to match.
